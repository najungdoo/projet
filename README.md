# 학생 QR 출결 관리 앱

학생마다 고유 QR 코드를 생성해 배부하고, 카메라 스캔으로 출결을 기록할 수 있는 웹 앱입니다.

## 기능

- 학생 목록 입력 (`학번,이름` 또는 `이름`)
- 학생별 QR 코드 자동 생성
- 카메라 기반 QR 스캔으로 당일 출석 체크
- 출결 현황 테이블 표시 (출석/미출석)
- 당일 출결 CSV 다운로드
- 브라우저 `localStorage` 기반 데이터 저장

## 실행 방법

별도 빌드 없이 정적 파일로 동작합니다.

### 방법 A) 파일 직접 열기

1. `index.html` 파일을 브라우저로 열기
2. 학생 정보를 입력하고 **QR 생성** 클릭
3. **스캐너 시작** 클릭 후 학생 QR 코드 스캔
4. 출결 현황 확인 및 CSV 다운로드

### 방법 B) 로컬 서버 실행 (권장)

카메라 권한/브라우저 보안정책으로 인해 로컬 서버 실행이 더 안정적입니다.

```bash
python3 -m http.server 8080
```

그 다음 브라우저에서 `http://localhost:8080` 접속 후 사용합니다.

> 카메라 권한 허용이 필요합니다.

## 테스트 방법 및 절차

아래 순서대로 테스트하면 핵심 기능을 빠르게 검증할 수 있습니다.

### 1) 사전 준비

- 카메라가 있는 기기(노트북/모바일) 준비
- Chrome/Edge/Safari 최신 버전 권장
- HTTPS 또는 `localhost` 환경에서 실행 (카메라 권한 문제 방지)

### 2) 기본 동작 테스트

1. 앱 접속 후 학생 입력란에 다음 예시 입력
   ```
   2026001,김민준
   2026002,이서연
   박지훈
   ```
2. **QR 생성** 클릭
3. 학생 목록 테이블에 학생 3명과 QR이 보이는지 확인
4. 출결 요약이 `총 3명`으로 표시되는지 확인

### 3) 스캔 출결 테스트

1. **스캐너 시작** 클릭
2. 같은 화면의 학생 QR을 다른 기기로 비추거나, 출력물 QR을 카메라에 비춤
3. 스캔 성공 시 상태 메시지(`출석 완료`)와 테이블 상태(`출석`) 확인
4. 같은 학생 QR을 다시 스캔하여 `이미 출석 처리` 메시지 확인
5. 미등록 QR 또는 일반 QR 스캔 시 오류 메시지 확인

### 4) 초기화/저장 테스트

1. 페이지 새로고침 후 학생/출결 정보가 유지되는지 확인 (`localStorage`)
2. **오늘 출결 초기화** 클릭 후 모든 학생이 `미출석`으로 바뀌는지 확인
3. **학생 목록 초기화** 클릭 후 학생 테이블이 비워지는지 확인

### 5) CSV 다운로드 테스트

1. 일부 학생을 출석 처리
2. **CSV 다운로드** 클릭
3. 생성된 `attendance-YYYY-MM-DD.csv` 파일 열기
4. 다음 컬럼 존재 여부 확인
   - `date, student_id, name, status, checked_at`
5. 출석 학생은 `present`, 미출석 학생은 `absent`로 저장되는지 확인

### 6) 코드 문법 체크 (개발자용)

```bash
node --check app.js
```

정상이라면 에러 없이 종료됩니다.

## 파일 구조

- `styles.css`: 스타일
- `app.js`: QR 생성, 스캔, 출결 로직
